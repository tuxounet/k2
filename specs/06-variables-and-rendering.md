# Variables and Template Rendering

## Description

k2 uses the Go template engine (`text/template`) enhanced by the [Sprig](https://masterminds.github.io/sprig/) library for file and script rendering.

## Variable System

### Variable Hierarchy

Variables are merged according to the following priority (last value wins):

1. **Template parameters** (`template.body.parameters`) — default values
2. **Apply variables** (`apply.body.vars`) — overrides parameters

The merge is performed by `MergeMaps()`:

```go
mergedVars := MergeMaps(template.K2.Body.Parameters, apply.K2.Body.Vars)
```

### Supported Variable Types

Apply variables (`body.vars`) support all YAML types:

| Type | YAML Example | Template Access |
|------|-------------|----------------------|
| String | `name: "value"` | `{{ .name }}` |
| List | `coll: [a, b, c]` | `{{ range .coll }}{{ . }}{{ end }}` |
| Object | `obj: {a: 1, b: stc}` | `{{ .obj.b }}` |
| Boolean | `flag: true` | `{{ .flag }}` |
| Number | `count: 42` | `{{ .count }}` |

> **Note**: Template parameters (`body.parameters`) are of type `map[string]any`, inventory variables (`body.vars`) are of type `map[string]string`.

## File Rendering

During apply, each template file is read, rendered with the merged variables, then written to the target folder.

### Go Template Syntax

```
{{ .variable }}              — Access a variable
{{ range .list }}...{{ end }} — Iterate over a list
{{ .object.field }}          — Access an object field
{{ if .condition }}...{{ end }} — Conditional
```

### Sprig Functions

All [Sprig functions](https://masterminds.github.io/sprig/) are available:

```
{{ .name | upper }}          — Uppercase
{{ .name | lower }}          — Lowercase
{{ .name | title }}          — First letter capitalized
{{ .name | quote }}          — Surrounded by quotes
{{ default "val" .name }}    — Default value
{{ .name | replace "a" "b" }} — Replacement
```

### Template File Example

Source file `README.md` from the `kind1` template:

```markdown
# {{ .name }}

{{ .description }}

## colls:
{{ range .coll }}
-  {{ . }}{{ end }}

## map data

{{ .obj.b }}
```

With the variables:
```yaml
vars:
  name: "component2"
  description: "Template of type kind1"
  coll:
    - a
    - b
    - c
  obj:
    a: 1
    b: stc
    c: true
```

Rendered result:
```markdown
# component2

Template of type kind1

## colls:

-  a
-  b
-  c

## map data

stc
```

## Script Rendering

Script commands are also rendered through the Go template engine before execution:

```yaml
scripts:
  bootstrap:
    - echo "boot {{ .name }}"
  post:
    - echo "template fin of {{ .name }}"
```

With `name: "component2"`, the executed command will be:
```bash
echo "boot component2"
```

## Rendering Exclusion

The `.gitignore` source file of a template is not copied directly: its content is merged into the `.gitignore` generated by k2 (see [09-gitignore-and-files.md](09-gitignore-and-files.md)).
