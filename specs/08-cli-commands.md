# CLI Commands

## Description

k2 exposes three commands via the CLI: `render-plan`, `render`, and `unrender`. Each command accepts an `--inventory` flag to specify the inventory file.

## Installation

```bash
go install github.com/tuxounet/k2@latest
```

## Common Flag

| Flag | Type | Default | Description |
|------|------|---------|-------------|
| `--inventory` | string | `./k2.inventory.yaml` | Path to the inventory file |

## `render-plan` Command

### Description

Analyzes the inventory and displays the execution plan without making any changes. Useful for previewing actions before a render.

### Usage

```bash
k2 render-plan [--inventory <path>]
```

### Behavior

1. Loads the inventory file
2. Scans folders looking for templates and applies
3. Creates the execution plan (resolutions + applications)
4. Deduplicates tasks
5. Displays the planned tasks

### Output

```
OUT: Planning inventory ./k2.inventory.yaml
OUT: Found Apply: 5
OUT: PLAN RESULT: 8
OUT: WILL DO ACTION: {local-resolve map[hash:abc123 params:...]}
OUT: WILL DO ACTION: {git-resolve map[hash:def456 params:...]}
OUT: WILL DO ACTION: {apply map[id:my-service ref:abc123]}
```

The displayed task types:
- `local-resolve`: resolution of a template from the inventory
- `git-resolve`: clone/update of a Git repository
- `apply`: application of a template

## `render` Command

### Description

Executes the render plan: resolves templates, copies files with rendering, runs lifecycle scripts.

### Usage

```bash
k2 render [--inventory <path>]
```

### Behavior

1. Loads the inventory and creates the plan (identical to `render-plan`)
2. Sequentially executes each task in the plan:
   - **local-resolve**: loads the template from the inventory
   - **git-resolve**: clones or updates the Git repository in `.refs/<hash>/`
   - **apply**: runs the complete cycle (bootstrap → pre → copy → post)
3. For each apply:
   - Detects if it's the first render (via `.gitignore`)
   - Runs bootstrap scripts (if first render)
   - Runs pre scripts
   - Copies and renders template files
   - Generates the `.gitignore`
   - Runs post scripts

### Idempotence

Render is **idempotent**: it can be run multiple times. Files are overwritten and the `.gitignore` is regenerated. Bootstrap scripts are only executed on the first render.

## `unrender` Command

### Description

Deletes all files generated by renders and cleans up Git references.

### Usage

```bash
k2 unrender [--inventory <path>]
```

### Behavior

1. Loads the inventory and creates the plan
2. For each `git-resolve` task: deletes the `.refs/<hash>/` folder
3. For each `apply` task:
   - Runs `template.scripts.post` (if the template is resolved)
   - Runs `apply.scripts.nuke`
   - Reads the `.gitignore` file from the target folder
   - Deletes each file/folder listed in the `.gitignore`
   - Deletes the `.gitignore` itself
4. Cleans up remaining empty folders (recursive upward traversal)

### Preserved Files

Entries starting with `!` in the `.gitignore` are ignored during deletion. This notably preserves `k2.apply.yaml` and custom files.

## Examples

```bash
# Preview actions
k2 render-plan

# Render templates
k2 render

# Unrender generated files
k2 unrender

# Use a specific inventory
k2 render --inventory ./my-inventory.yaml
```
